# NoteHaven: Database Architecture Review & Build Verification

## Task 1: Database Architecture Investigation

### Current State: Supabase is the ONLY active database

MongoDB is **no longer used**. The migration is complete. The old architecture was:

```
BEFORE:  Frontend --> Express Backend (localhost:3001) --> MongoDB (media cache) + Supabase (auth/data)
AFTER:   Frontend --> Supabase Edge Function --> Supabase DB (everything)
```

**Supabase handles:** Auth, notes, tasks, prompts, media tracker, tags, ledger, subscriptions, birthdays, countdowns, shared notes, code snippets, user preferences, and media metadata caching (653+ items).

### Residual MongoDB References (Dead Code)

| Item | Location | Issue |
|------|----------|-------|
| `MONGODB_URI` with credentials | `.env:16` | **Security risk** - exposed username/password |
| Legacy backend vars | `.env:12-13,22-27` | Unused: `VITE_API_URL`, `PORT`, `ALLOWED_ORIGINS`, `RATE_LIMIT_*` |
| `fetchImagesFromMongoDB` alias | `src/lib/simple-image-fetcher.ts:283` | Misleading name, actually calls Supabase |
| Import of `fetchImagesFromMongoDB` | `src/pages/MediaTracker.tsx:45,145,266,290` | Should be renamed |
| Comment "simple MongoDB-only fetch" | `src/pages/MediaTracker.tsx:145` | Outdated |
| `_id` field in `ExternalMedia` | `src/lib/media-api.ts:11` | MongoDB naming convention remnant |
| `MIGRATION_COMPLETE.md` / `MIGRATION_GUIDE.md` | Root | Historical docs, can be archived |

### Recommended Cleanup

1. Remove `MONGODB_URI` and legacy backend vars from `.env`
2. Rename `fetchImagesFromMongoDB` -> `fetchImagesFromSupabase` in `simple-image-fetcher.ts` and all callers in `MediaTracker.tsx`
3. Update outdated MongoDB comment in `MediaTracker.tsx:145`
4. Remove or rename `_id` field in `media-api.ts`
5. Archive/remove migration docs

## Task 2: Dashboard Refactoring Verification

### Run Locally

```bash
bun run dev
# Starts Vite dev server at http://localhost:8080
```

### Build Result: TypeScript PASS, PWA FAIL

- All **2,670 modules compile successfully** - no TypeScript or import errors
- Build fails at PWA service worker generation: bundle is **2.35 MB** (Workbox default limit is 2 MB)
- This is a **pre-existing issue**, not caused by the dashboard refactoring

**Fix for build:** Add to `vite.config.ts`:
```ts
workbox: {
  maximumFileSizeToCacheInBytes: 3 * 1024 * 1024, // 3 MB
}
```

### Lint: 63 problems (only 2 are dashboard-related)

| File | Line | Issue |
|------|------|-------|
| `Dashboard.tsx` | 214 | Warning: `useCallback` missing `renewals` dependency |
| `Dashboard.tsx` | 786 | Error: `@typescript-eslint/no-explicit-any` on `user_metadata` cast |

### Dashboard Code Bugs Found

1. **Stale `renewals` in calendar events** (`Dashboard.tsx:198`) - `generateCalendarEvents()` uses the state variable `renewals` from the previous render, not the freshly fetched value. Calendar events will always be one render behind.

2. **WidgetManager state sync** (`WidgetManager.tsx:55`) - `localWidgets` initialized from props but never re-synced when props change. Dialog shows stale data.

3. **Unsafe `any` cast** (`Dashboard.tsx:786`) - `(user?.user_metadata as any)` should use proper typing.
